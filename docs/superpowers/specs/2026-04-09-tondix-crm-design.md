# Tondix CRM — Document de design

**Date** : 2026-04-09  
**Approche retenue** : B — Catalogue produits + extension des deals  
**Stack** : Atomic CRM (React + Supabase), fork `tondix-crm-multiclaude`

---

## Contexte

Tondix est un vendeur de tondeuses B2B. L'équipe commerciale est composée de 2 à 5 personnes. La clientèle est mixte : indépendants (paysagistes), PME (espaces verts), et grandes structures (collectivités, golfs, stades). Le catalogue comprend 20 à 100 références. Deux activités principales : **vente de tondeuses** et **contrats d'entretien annuels**.

---

## 1. Modèle de données

### Tables existantes (réutilisées sans modification)

| Table | Usage Tondix |
|---|---|
| `contacts` | Interlocuteurs chez les clients |
| `companies` | Clients B2B (golfs, mairies, paysagistes…) |
| `deals` | Affaires de vente de tondeuses |
| `tasks` | Suivi des actions commerciales |
| `contact_notes` / `deal_notes` | Historique des échanges |
| `sales` | Membres de l'équipe Tondix |
| `tags` | Étiquettes libres |

### Nouvelles tables

#### `products` — Catalogue tondeuses

```sql
create table public.products (
    id bigint generated by default as identity primary key,
    created_at timestamp with time zone not null default now(),
    reference text not null,          -- ex: "TX-450"
    name text not null,               -- ex: "Tondix Pro 450"
    type text,                        -- thermique | electrique | robot | autoportee | accessoire
    price bigint,                     -- prix catalogue en centimes (EUR)
    description text,
    active boolean not null default true,
    sales_id bigint references public.sales(id)
);
```

#### `deal_products` — Produits dans une affaire

```sql
create table public.deal_products (
    id bigint generated by default as identity primary key,
    deal_id bigint not null references public.deals(id) on delete cascade,
    product_id bigint not null references public.products(id),
    quantity smallint not null default 1,
    unit_price bigint                 -- prix négocié en centimes (peut différer du catalogue)
);
```

#### `service_contracts` — Contrats d'entretien annuels

```sql
create table public.service_contracts (
    id bigint generated by default as identity primary key,
    created_at timestamp with time zone not null default now(),
    name text not null,
    company_id bigint references public.companies(id) on delete cascade,
    contact_ids bigint[],
    deal_id bigint references public.deals(id),  -- vente associée (optionnel)
    start_date date not null,
    renewal_date date not null,
    amount bigint,                    -- montant annuel en centimes
    notes text,
    status text not null default 'actif',  -- actif | a-renouveler | resilier
    sales_id bigint references public.sales(id)
);
```

## 4. Nouvelles pages

### Catalogue tondeuses (`/products`)

- **Liste** : tableau avec référence, nom, type, prix catalogue, statut actif/inactif
- **Fiche** : détail produit + historique des affaires où il apparaît
- **Formulaire** : création / édition d'un produit

### Contrats d'entretien (`/service_contracts`)

- **Liste** : tableau avec client, date de renouvellement, montant, statut — triée par renouvellement le plus proche
- **Fiche** : détail contrat + lien vers la vente d'origine (si renseignée)
- **Formulaire** : création / édition

### Extension de la fiche Affaire (`/deals`)

Ajout d'un panneau "Produits" dans la fiche d'une affaire :
- Sélection de produits du catalogue
- Quantité et prix négocié par ligne
- Total calculé automatiquement

## 5. Dashboard — Widget renouvellements

Ajout d'un widget "Contrats à renouveler" sur le tableau de bord existant :
- Liste des contrats dont `renewal_date` est dans les 60 prochains jours
- Lien direct vers la fiche contrat

## 6. Périmètre exclu

- Génération de devis PDF (hors scope)
- Gestion des stocks / inventaire (hors scope)
- Tarification différenciée par segment client (hors scope)
- Application mobile (non prioritaire)
